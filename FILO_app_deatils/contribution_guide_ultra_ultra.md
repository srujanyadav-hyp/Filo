
# FILO — ULTRA–ULTRA EXPANDED CONTRIBUTION GUIDE

This file defines EXACT rules for contributing to FILO.

---

# 1. BEFORE CONTRIBUTING
- Read onboarding guide  
- Read coding standards  
- Understand rule engine data flow  
- Run all tests  

---

# 2. CREATING A FEATURE

### Steps:
1. Write problem statement  
2. Write acceptance criteria  
3. Write test plan  
4. Implement feature  
5. Document action  

### PR Requirements:
- Must include screenshots  
- Must include test results  
- Must include architecture note  

---

# 3. WHAT IS NOT ALLOWED
- Adding new dependencies without approval  
- Writing untyped JSON  
- Mutating state outside Riverpod  
- Hardcoding paths  
- Mixing UI + logic  

---

# 4. REVIEW PROCESS
Your PR will be evaluated for:
- logic correctness  
- safety  
- architecture consistency  
- performance  
- maintainability
